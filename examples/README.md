# Examples · 真实成片示范库(v5.6 新增 · 借鉴 hugohe3/ppt-master examples/)

## 这里放什么

**脱敏后的真实成片**——给 AI(以及饶秋自己)看"什么是真正合格的成品"。

借鉴 PPT Master 的 22 个示范 deck 共 309 页的理念:**SKILL.md 讲规则,examples 讲手感**。规则可以写得很硬,但 AI 写出来什么样还是要有真片对照。

## 目录结构(占位 · 按需填)

```
examples/
├── README.md                       ← 你正在看的这份
├── training/                       ← 培训片示范
│   └── _placeholder.md
├── client-proposal/                ← 客户提案示范
│   └── _placeholder.md
├── laimei/                         ← 莱美对外文档示范
│   └── _placeholder.md
└── xhs-or-self-media/              ← 自媒体单页示范(若适用)
    └── _placeholder.md
```

每个子目录里放:
- `<场景名>-脱敏版.html` — 成片
- `<场景名>-说明.md` — 这份做了什么 / 用了哪些版式 / 为什么这样选(后人读得懂)

## 脱敏规则(P0 红线 · 不可妥协)

放进 examples/ 之前**必须脱敏**,违反任意一条不许放:

| 规则 | 做法 |
|---|---|
| 客户真实名称 | 替换为"客户 A / B / C" |
| 学员真实姓名 | 替换为"学员 1 / 2 / 3" |
| 团队真名(秦卓 / 熊玉婷 / 刘东琴 / 曾正) | 删掉或替换为角色名 |
| 莱美内部订单 / 双轨制定价 | 删掉或改为虚构数字(标注"示意,非真实数据") |
| 客户门店名(亨得利 / 爱成眼科) | 替换为"客户门店 A" |
| 健康具体数值 / 家族肿瘤史 / OSA | 全部删,不出现 |
| 陶勇医生 / 迪士尼镜片×陶勇团队 | 完全删,不允许出现 |
| 手机号 / 身份证 / 地址精确门牌 | 删 |
| 真实订单金额 | 改为示意值,标注"示意" |

**自检命令**(放成片到 examples/ 之前跑一遍):

```bash
grep -i "秦卓\|熊玉婷\|刘东琴\|曾正\|陶勇\|亨得利\|爱成眼科" examples/**/*.html
# 期望:0 匹配
```

## 填这个目录的时机

**不强求一次填满**。原则是:
- 每做完一份**自己觉得很满意**的真实片,留个 5 分钟做脱敏,扔进 examples
- 优先级:培训片 > 客户提案 > 莱美 > 自媒体(因为饶秋培训片最多,样本最有用)
- 每个子目录有 1-3 份示范就够了,**不需要堆量**

## AI 怎么用这个目录(Mode A 工作流补充)

Mode A 接到任务后,**如果对应子目录有示范片**:
1. 先 `Read` 一份对应场景的示范片(`training/` 看培训示范、`client-proposal/` 看提案示范)
2. 学习它的:
   - 版式选择(哪些页用 cover / chapter / card-grid / metric+split / dashboard)
   - 节奏(hero 页和 info 页的交替)
   - 字号 / 间距 / 留白的体感
   - 标题写作风格(动词 / 名词性 / 长短)
3. 再开始写新片

**这比读 SKILL.md 抽象规则有效得多。**

如果对应子目录为空 → 跳过这一步,按 SKILL.md 默认流程走。
